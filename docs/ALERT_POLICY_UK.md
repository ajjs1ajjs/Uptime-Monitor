# Політика сповіщень та перевірок

## Як змінити налаштування

### Через веб-інтерфейс
1. Перейдіть на вкладку **⚙️ Налаштування**
2. Знайдіть блок **⚡ Політика сповіщень та перевірок**
3. Змініть потрібні параметри
4. Натисніть **💾 Зберегти політику сповіщень**

### Через файл config.json
1. Відкрийте файл конфігурації:
   - Linux: `/etc/uptime-monitor/config.json`
   - Windows: `%USERPROFILE%\UptimeMonitor\config.json`
2. Знайдіть секцію `"alert_policy"` (або створіть її)
3. Додайте/змініть потрібні параметри
4. Збережіть файл — зміни підхопляться автоматично протягом 30 секунд

Приклад:
```json
{
  "alert_policy": {
    "request_timeout_seconds": 30,
    "grace_period_seconds": 60,
    "up_success_threshold": 2,
    "still_down_repeat_seconds": 600,
    "treat_4xx_as_down": true,
    "ssl_notification_days": [30, 14, 7, 5, 3, 1],
    "ssl_notification_cooldown_seconds": 21600,
    "ssl_check_interval_hours": 6,
    "verify_ssl": true,
    "retry_delays": [30, 30],
    "max_retries": 2,
    "flapping_threshold": 3,
    "flapping_window_seconds": 300,
    "flapping_suppression_seconds": 600
  }
}
```

---

## Опис параметрів

### Перевірки

| Параметр | Дефолт | Мін | Макс | Опис |
|----------|--------|-----|------|------|
| `request_timeout_seconds` | 30 | 1 | 300 | Максимальний час очікування відповіді HTTP/пінгу |
| `retry_delays` | [30, 30] | 1 | 300 | Затримки між повторами при невдалій перевірці (сек) |
| `max_retries` | 2 | 0 | 10 | Кількість повторних спроб перед фіксацією DOWN |
| `up_success_threshold` | 2 | 1 | 20 | Скільки успішних перевірок поспіль для відновлення статусу UP |

### Сповіщення про DOWN

| Параметр | Дефолт | Мін | Макс | Опис |
|----------|--------|-----|------|------|
| `grace_period_seconds` | 0 | 0 | 3600 | Затримка перед відправкою першого сповіщення про DOWN. Наприклад, 60 = зачекати 60 сек після виявлення проблеми, перш ніж сповіщати. 0 = сповіщати одразу |
| `still_down_repeat_seconds` | 600 | 60 | 86400 | Через скільки секунд повторювати сповіщення, якщо сайт досі недоступний |
| `treat_4xx_as_down` | true | — | — | Якщо true — HTTP 4xx помилки вважаються DOWN. Якщо false — DOWN тільки 5xx |

### Флапінг (хитання статусу)

Флапінг — це часта зміна статусу UP↔DOWN за короткий проміжок часу. Система виявляє флапінг і автоматично придушує сповіщення, щоб уникнути спаму.

| Параметр | Дефолт | Мін | Макс | Опис |
|----------|--------|-----|------|------|
| `flapping_threshold` | 3 | 2 | 20 | Скільки змін статусу за вікно для активації придушення |
| `flapping_window_seconds` | 300 | 60 | 3600 | Вікно часу, в якому рахуються зміни статусу (сек) |
| `flapping_suppression_seconds` | 600 | 60 | 7200 | Скільки секунд придушувати сповіщення після виявлення флапінгу |

**Приклад:** при дефолтних значеннях, якщо монітор змінить статус 4 рази за 5 хвилин, сповіщення будуть придушені на 10 хвилин. Через 10 хвилин стабільного стану флапінг скидається.

### SSL сертифікати

| Параметр | Дефолт | Мін | Макс | Опис |
|----------|--------|-----|------|------|
| `ssl_notification_days` | [30,14,7,5,3,1] | 1 | — | Пороги днів до закінчення SSL, при досягненні яких надсилається сповіщення. Можна вказати через кому в UI |
| `ssl_notification_cooldown_seconds` | 21600 (6 год) | 300 | 86400 | Мінімальний інтервал між SSL сповіщеннями |
| `ssl_check_interval_hours` | 6 | 1 | 168 | Як часто перевіряти SSL сертифікати |
| `verify_ssl` | true | — | — | Перевіряти валідність SSL сертифіката при HTTP-запитах |

## Міграція зі старої конфігурації

Якщо у вашому `config.json` був параметр `down_failures_threshold`, система автоматично конвертує його в `grace_period_seconds` за формулою:
```
grace_period_seconds = down_failures_threshold × 30
```

Старий параметр `ssl_notification_days: 14` (число) автоматично конвертується в `ssl_notification_days: [14]`.

## Зміна налаштувань монітора через UI

Деякі параметри налаштовуються індивідуально для кожного монітора при створенні/редагуванні:
- **Інтервал перевірки** — Edit Monitor → Check Interval (сек)
- **Методи сповіщень** — Edit Monitor → вибір каналів
- **Ключове слово** — для HTTP моніторів (опціонально)
